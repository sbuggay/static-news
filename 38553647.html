Findecanor|root|parent|next|

I don't think Bitmanip is special but pretty much required for reaching feature-parity with ARM and x86 so that you could start comparing them. RISC-V is otherwise quite spartan, which might be desirable for embedded MCUs, but for application-class processors you'd want more. For example, an indexed load which on ARM and x86 is one instruction could on RISC-V require four dependent instructions without Bitmanip and two with.

The vector extension is RISC-V's closest counterpart to SSE/AVX, although it is not used for scalar floating-point.

BTW, the RISC-V foundation has published official Application Profiles for application-class processors. Those are probably going to be baseline for binary distributions in the future — they include Bitmanip and more. The RVA23 profile has Vector. Google's requirements for Android (with Google services) will also include vector cryptography.

brucehoult|root|parent|

Performance per core is the same as (even a little better than) Arm A72, as found in the Raspberry Pi 4 or -- more appropriately -- the Graviton 1, both from 2019, except it has 16x more cores than the Pi 4 and 4x more cores than the Graviton.

That puts shipping RISC-V hardware four years behind Arm.

By mid 2024, shipping RISC-V hardware will be competitive with Arm A76 SoCs such as the RK3588 (shipped on boards such as the Rock 5 and Orange Pi 5 1.5 to 2 years earlier) or the Raspberry Pi 5, which is only about 6 months ahead (I received mine, ordered at announcement, this week).

By the end of 2024, shipping RISC-V hardware will have 16x Arm A78 equivalent cores, on a board that realistically will cost $300.
