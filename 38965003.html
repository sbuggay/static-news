Tried it for about 5 minutes and are the issues I ran into in that time:

- colors don't work with my terminal's default light theme, have to set ENV var telling it I have a light theme... Just like the first thing that happens when I install your software and press the button to launch it is it telling me "not so fast cowboy, go read my docs"

- bar with commands is lame, I don't want user interfaces in my terminal. have to set an ENV var to hide.

- my bash history format is two lines: a commented out timestamp followed by the command. This is configured using the "HISTFORMAT" environment variable. McFly doesn't use this variable, which leads to 2 issues: 1) when you search for commands it shows you how long ago each search result was run. It displays "3m" next to all my commands because it thinks all commands were ran the instant I installed McFly, but the timestamp is right there 2) when it writes commands to your bash history it doesn't include the timestamp comment so when you uninstall McFly, bash thinks all the commands McFly wrote to the history was a single multiline command when you press ctrl-p.

- when I press ctrl-p (or back arrow presumably), then it shows my entire bash history as the previous command

Overall, I didn't like it and I'm probably not going to try it ever again. I don't want to maintain a config for one keypress.
