There’s been a bunch of progress in the field since then. Oliver de Weck from MIT and his group have devised a way to calculate system complexity, correlates that with both cost and cognitive effort and shown, that it indeed has a hump-like shape on simulated systems. Furthermore, one can demonstrate, that in some sense optimal number of connections between system elements exists and that this optimum is linear in relation to the number of nodes for k>10. Thus, you can calculate the number of connections between your microservices beyond which you are better off with a monolith.
