st_goliath|parent|

System calls need to switch into kernel mode and back, this can be a massive performance hit.

This is especially bad if you want to do precise time measurements, a chunk of time is now spent just calling `gettimeofday`. The performance impact (benchmarked and shown in the article) is substantial.

Linux (the kernel) at some point added a "vdso", basically a shared library automatically mapped into every process. The libc can call into the vdso version of `gettimeofday` and use the system call as a fallback.

A 64 bit kernel that can run 32 bit programs, will need a 64 bit and 32 bit vdso.

On x86, the kernel build process can "simply" use the same compiler to produce 32 bit code for the 32 bit vdso. GCC for ARM can't, for some reason. You need 2 toolchains, a 64 bit and a 32 bit one.

If you don't know that, the kernel kernel only has a 64 bit vdso, the 32 bit program will run but will silently use the system call, instead of the vdso, causing unexpected performance issues.
